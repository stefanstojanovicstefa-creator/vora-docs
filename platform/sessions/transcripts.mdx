# Transcripts

Every conversation is automatically transcribed, providing a complete text record of agent-user interactions.

---

## Transcript Overview

A transcript is a text representation of a conversation:

```
[10:30:45] Agent: Hi! Thanks for calling Acme Corp. I'm Sarah,
                  your virtual assistant. How can I help you today?

[10:30:52] User:  Yeah, I need to know when my package is arriving.

[10:30:56] Agent: I'd be happy to help you track your order.
                  Could you provide your order number or the email
                  address on your account?

[10:31:05] User:  It's john@example.com

[10:31:08] Agent: Thank you. Let me look that up for you.
           [Function: order_lookup executed]

[10:31:12] Agent: I found your order. It shipped yesterday and is
                  expected to arrive by Friday, January 17th.
```

## Viewing Transcripts

### In the Dashboard

1. Go to **Sessions** ‚Üí Select a session
2. Click the **Transcript** tab
3. View the full conversation

### Transcript Features

| Feature | Description |
|---------|-------------|
| **Timestamps** | Time of each utterance |
| **Speaker labels** | Agent vs User identification |
| **Function calls** | Inline function executions |
| **Sentiment** | Per-message sentiment |
| **Confidence** | Transcription confidence scores |
| **Search** | Find specific text |

## Transcript Format

### Standard Format

Default transcript display:

```
[HH:MM:SS] Speaker: Message content
```

### Detailed Format

Includes additional metadata:

```
[10:30:45] Agent (confidence: 0.98, sentiment: 0.7):
           Hi! Thanks for calling Acme Corp.

[10:30:52] User (confidence: 0.95, sentiment: 0.4):
           I need to know when my package is arriving.
```

### JSON Format

Structured data for programmatic access:

```json
{
  "transcript": [
    {
      "timestamp": "2024-01-15T10:30:45Z",
      "speaker": "agent",
      "text": "Hi! Thanks for calling Acme Corp.",
      "confidence": 0.98,
      "sentiment": 0.7,
      "duration_ms": 2500
    },
    {
      "timestamp": "2024-01-15T10:30:52Z",
      "speaker": "user",
      "text": "I need to know when my package is arriving.",
      "confidence": 0.95,
      "sentiment": 0.4,
      "duration_ms": 3200
    }
  ]
}
```

## Transcript Search

### Within a Session

Search within a single transcript:

```
/order
```

Highlights all occurrences of "order" in the transcript.

### Across All Sessions

Search transcripts across all sessions:

1. Go to **Sessions** ‚Üí **Search**
2. Enter your query with `transcript:` prefix

```
transcript:"shipping delay"
```

### Search Operators

| Operator | Example | Matches |
|----------|---------|---------|
| `""` | `"order status"` | Exact phrase |
| `AND` | `order AND refund` | Both words |
| `OR` | `shipping OR delivery` | Either word |
| `NOT` | `order NOT cancelled` | Excludes word |
| `*` | `ship*` | Wildcard |

## Editing Transcripts

### Why Edit?

- Correct transcription errors
- Redact sensitive information
- Add context or notes
- Mark important sections

### Manual Corrections

1. Click on a message
2. Select **Edit**
3. Make corrections
4. Save changes

Original text is preserved with edit history.

### Redaction

Remove sensitive information:

1. Select the text to redact
2. Click **Redact**
3. Choose replacement: `[REDACTED]`, `[PHONE]`, `[EMAIL]`, etc.

```
[10:31:05] User: My credit card number is [REDACTED]
```

<Note>
  Redaction is permanent and cannot be undone. Original audio may still contain the sensitive information.
</Note>

## Real-Time Transcription

### Streaming Transcripts

Receive transcripts in real-time via webhook:

```javascript
// Webhook payload for transcript event
{
  "event": "transcript.updated",
  "sessionId": "sess_abc123",
  "data": {
    "speaker": "user",
    "text": "When will my order arrive?",
    "timestamp": "2024-01-15T10:30:52Z",
    "isFinal": true
  }
}
```

### Interim vs Final

| Type | Description |
|------|-------------|
| **Interim** | In-progress transcription (may change) |
| **Final** | Confirmed transcription (stable) |

Configure webhooks in **Settings** ‚Üí **Webhooks**.
