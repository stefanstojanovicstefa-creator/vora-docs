# Providers Overview

Vora connects to best-in-class AI providers for each stage of the voice pipeline. You can use Vora's defaults with no setup, or bring your own API keys for custom configuration.

---

<Tabs>
  <Tab title="Simple">
    ## How Providers Work

    Every voice AI conversation uses three types of providers:

    1. **Speech-to-Text (STT)** — Converts what the caller says into text
    2. **AI Model (LLM)** — Understands the text and generates a response
    3. **Text-to-Speech (TTS)** — Converts the response back to speech

    Vora picks the best providers by default. You don't need to configure anything.

    ### Default Providers

    | Component | Default | Why |
    |-----------|---------|-----|
    | **STT** | Deepgram | Fast and accurate for real-time voice |
    | **LLM** | Gemini Flash | Quick responses, broad language support |
    | **TTS** | ElevenLabs | Natural-sounding voices |

    <Tip>
      The defaults work great for most agents. You only need to change providers if you have specific language or performance requirements.
    </Tip>

    ## Provider Architecture

    ```
    Caller Audio → STT Provider → LLM Provider → TTS Provider → Agent Audio
    ```

    Each provider is independently configurable per agent in the [Command Center](/advanced/command-center).

    ## Configuring Providers

    Configure per agent in the [Command Center](/advanced/command-center):
    - **Ears Tab** → STT provider and language
    - **Brain Tab** → LLM provider and model
    - **Voice Tab** → TTS provider and voice

    ## Bring Your Own Keys

    Use your own provider API keys for higher rate limits and direct billing. See [API Keys](/advanced/api-keys).
