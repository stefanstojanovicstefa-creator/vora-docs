## Usage Alerts

### Configure Alerts

Set up notifications to stay informed:

```javascript
{
  "usageAlerts": {
    "thresholds": [
      { "percent": 50, "notify": true },
      { "percent": 80, "notify": true },
      { "percent": 100, "notify": true }
    ],
    "budgetLimit": {
      "amount": 150,
      "currency": "USD",
      "notify": true
    },
    "channels": {
      "email": ["billing@company.com"],
      "slack": "#billing-alerts"
    }
  }
}
```

### Alert Types

| Alert | Trigger | Action |
|-------|---------|--------|
| **50% Used** | Half minutes consumed | Informational |
| **80% Used** | Approaching limit | Review usage |
| **100% Used** | Entering overage | Consider upgrade |
| **Budget Limit** | Cost threshold reached | Hard stop optional |

### Hard Limits (Optional)

Prevent overage charges entirely:

```javascript
{
  "hardLimits": {
    "enabled": true,
    "action": "pause_new_sessions",
    "threshold": "included_minutes",  // or specific number
    "notify": true,
    "allowExceptions": ["agent_xyz"]  // Emergency agent
  }
}
```

## Usage API

### Get Current Usage

```javascript
const usage = await vora.billing.usage.get({
  period: 'current'
});

console.log(usage);
// {
//   period: { start: '2024-01-01', end: '2024-01-31' },
//   minutes: {
//     used: 347,
//     included: 500,
//     remaining: 153,
//     projected: 485
//   },
//   sessions: {
//     total: 891,
//     inbound: 412,
//     outbound: 156,
//     web: 234,
//     api: 89
//   },
//   cost: {
//     base: 99,
//     overage: 0,
//     addons: 4,
//     projected: 103
//   }
// }
```

### Get Historical Usage

```javascript
const history = await vora.billing.usage.history({
  periods: 6,  // Last 6 months
  granularity: 'month'
});

for (const period of history) {
  console.log(`${period.name}: ${period.minutes} minutes, $${period.cost}`);
}
```

### Get Usage by Agent

```javascript
const agentUsage = await vora.billing.usage.byAgent({
  period: 'current'
});

for (const agent of agentUsage) {
  console.log(`${agent.name}: ${agent.minutes} minutes (${agent.percent}%)`);
}
```

### Export Usage Data

```javascript
const csv = await vora.billing.usage.export({
  period: 'current',
  format: 'csv',
  include: ['sessions', 'agents', 'daily']
});

fs.writeFileSync('usage-report.csv', csv);
```

## Usage Reports

### Scheduled Reports

Configure automatic reports:

1. Go to **Settings** → **Reports**
2. Click **Create Report**
3. Select usage metrics
4. Choose frequency (daily, weekly, monthly)
5. Add recipients

### Report Types

| Report | Contents | Frequency |
|--------|----------|-----------|
| **Daily Summary** | Sessions, minutes, peak times | Daily |
| **Weekly Digest** | Trends, comparisons, alerts | Weekly |
| **Monthly Invoice** | Full billing breakdown | Monthly |
| **Agent Performance** | Per-agent metrics | Weekly |

### Custom Reports

Build custom reports with specific metrics:

```javascript
const report = await vora.reports.create({
  name: 'Weekly Agent Performance',
  metrics: [
    'sessions_count',
    'avg_duration',
    'resolution_rate',
    'transfer_rate'
  ],
  groupBy: 'agent',
  period: 'last_7_days',
  schedule: {
    frequency: 'weekly',
    day: 'monday',
    time: '09:00'
  },
  recipients: ['team@company.com']
});
```

## FAQs

<AccordionGroup>
  <Accordion title="How are minutes calculated?">
    Minutes are the total voice session duration, rounded to the nearest second. Only voice (audio) sessions count—text chats and API calls don't use minutes.
  </Accordion>

  <Accordion title="Why is my projected usage higher than expected?">
    Projections are based on current daily averages extrapolated to end of period. Weekends and holidays may not be factored in, leading to higher projections.
  </Accordion>

  <Accordion title="Can I see individual session details?">
    Yes, go to **Sessions** to see every session with full transcripts, recordings, and metadata.
  </Accordion>

  <Accordion title="How can I reduce my usage?">
    Focus on: shorter agent responses, efficient prompts, knowledge base coverage, and routing simple queries to self-service.
  </Accordion>
</AccordionGroup>
