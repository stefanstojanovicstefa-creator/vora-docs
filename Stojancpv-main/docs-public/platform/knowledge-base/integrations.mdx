# Knowledge Base Integrations

Connect your knowledge base to external platforms and keep content automatically synchronized.

---

## Available Integrations

| Platform | Sync Direction | Real-time | Notes |
|----------|----------------|-----------|-------|
| [**Notion**](#notion) | Pull | Webhooks | Pages, databases |
| [**Confluence**](#confluence) | Pull | Polling | Spaces, pages |
| [**Google Drive**](#google-drive) | Pull | Webhooks | Docs, Sheets |
| [**SharePoint**](#sharepoint) | Pull | Polling | Documents, lists |
| [**Intercom**](#intercom) | Pull | Webhooks | Articles, FAQs |
| [**Zendesk**](#zendesk) | Pull | Webhooks | Help center |
| [**GitHub**](#github) | Pull | Webhooks | Markdown files |

## Notion

Sync content from Notion workspaces.

### Setup

<Steps>
  <Step title="Connect Notion">
    Go to **Knowledge Base** → **Integrations** → **Notion** → **Connect**.
  </Step>
  <Step title="Authorize">
    Select which pages and databases Vora can access.
  </Step>
  <Step title="Configure Sync">
    Choose what to sync:
    - Specific pages
    - Entire databases
    - Workspaces
  </Step>
  <Step title="Set Schedule">
    Choose sync frequency (real-time via webhooks or scheduled).
  </Step>
</Steps>

### What's Synced

| Content Type | Synced | Notes |
|--------------|--------|-------|
| Page content | ✓ | Full text including nested blocks |
| Databases | ✓ | Each row becomes a chunk |
| Images | ✗ | Not extracted |
| Embedded content | Partial | Depends on embed type |
| Comments | ✗ | Not synced |

### Configuration Options

```javascript
{
  "notion": {
    "workspaceId": "workspace_123",
    "sync": {
      "pages": ["page_id_1", "page_id_2"],
      "databases": ["db_id_1"],
      "includeSubpages": true,
      "excludePatterns": ["**/Archive/**"]
    },
    "schedule": "realtime"
  }
}
```

## Confluence

Sync documentation from Atlassian Confluence.

### Setup

<Steps>
  <Step title="Connect Confluence">
    Go to **Knowledge Base** → **Integrations** → **Confluence** → **Connect**.
  </Step>
  <Step title="Authenticate">
    Enter your Confluence URL and API token.
  </Step>
  <Step title="Select Spaces">
    Choose which spaces to sync.
  </Step>
  <Step title="Configure Options">
    Set sync frequency and filtering options.
  </Step>
</Steps>

### What's Synced

| Content Type | Synced | Notes |
|--------------|--------|-------|
| Pages | ✓ | Full content |
| Blog posts | Optional | Can include or exclude |
| Attachments | Partial | PDFs and documents extracted |
| Comments | ✗ | Not synced |
| Labels | ✓ | Used as metadata |

### Configuration Options

```javascript
{
  "confluence": {
    "siteUrl": "https://yourcompany.atlassian.net",
    "spaces": ["DOCS", "SUPPORT"],
    "includeBlogs": false,
    "labelFilters": ["public", "kb"],
    "excludeLabels": ["draft", "internal"],
    "schedule": "daily"
  }
}
```

## Google Drive

Sync documents from Google Drive.

### Setup

<Steps>
  <Step title="Connect Google">
    Go to **Knowledge Base** → **Integrations** → **Google Drive** → **Connect**.
  </Step>
  <Step title="Authorize">
    Grant Vora access to your Google Drive.
  </Step>
  <Step title="Select Folders">
    Choose specific folders or shared drives to sync.
  </Step>
  <Step title="Configure">
    Set file types and sync schedule.
  </Step>
</Steps>

### Supported File Types

| File Type | Synced | Notes |
|-----------|--------|-------|
| Google Docs | ✓ | Full content extracted |
| Google Sheets | ✓ | Converted to structured text |
| Google Slides | ✓ | Text and speaker notes |
| PDFs | ✓ | Text extracted |
| Word/Excel | ✓ | Converted and extracted |
| Images | ✗ | Not processed |

### Configuration Options

```javascript
{
  "googleDrive": {
    "folders": ["folder_id_1", "folder_id_2"],
    "sharedDrives": ["drive_id_1"],
    "includeSubfolders": true,
    "fileTypes": ["document", "spreadsheet", "pdf"],
    "excludePatterns": ["**/Archive/**"],
    "schedule": "weekly"
  }
}
```

## GitHub

Sync documentation from GitHub repositories.

### Setup

<Steps>
  <Step title="Connect GitHub">
    Go to **Knowledge Base** → **Integrations** → **GitHub** → **Connect**.
  </Step>
  <Step title="Authorize">
    Install the Vora GitHub App on your organization.
  </Step>
  <Step title="Select Repositories">
    Choose which repositories to sync.
  </Step>
  <Step title="Configure Paths">
    Specify which files/folders to include.
  </Step>
</Steps>

### What's Synced

| Content Type | Synced | Notes |
|--------------|--------|-------|
| Markdown files | ✓ | .md, .mdx files |
| README files | ✓ | Automatically included |
| Documentation | ✓ | /docs folders |
| Code files | Optional | With comments |
| Wiki pages | Optional | Repository wikis |

### Configuration Options

```javascript
{
  "github": {
    "repositories": ["owner/repo1", "owner/repo2"],
    "branches": ["main", "docs"],
    "paths": ["docs/**", "*.md"],
    "excludePaths": ["**/node_modules/**"],
    "includeCode": false,
    "includeWiki": true,
    "schedule": "realtime"
  }
}
```

## Error Handling

### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| **Auth failed** | Token expired | Reconnect integration |
| **Permission denied** | Access revoked | Re-authorize with proper scopes |
| **Rate limited** | Too many requests | Wait and retry |
| **Content too large** | File size limit | Exclude or split large files |
| **Not found** | Content deleted | Will be removed on next sync |

### Reconnecting

If an integration disconnects:

1. Go to **Integrations** → Find the integration
2. Click **Reconnect**
3. Re-authorize access
4. Trigger a full sync

## Best Practices

<AccordionGroup>
  <Accordion title="Use selective sync">
    Don't sync everything. Select only content that should be available to your agent.
  </Accordion>

  <Accordion title="Enable real-time when available">
    For frequently updated content, use webhooks instead of scheduled syncs.
  </Accordion>

  <Accordion title="Monitor sync health">
    Regularly check sync status and address errors promptly.
  </Accordion>

  <Accordion title="Use labels and metadata">
    Organize content with labels in your source platform for better filtering.
  </Accordion>

  <Accordion title="Test after major changes">
    When you restructure content in the source, test retrieval in your knowledge base.
  </Accordion>
</AccordionGroup>
