# المفاهيم الأساسية

قبل بناء أول وكيل صوتي لك، إليك المفاهيم الرئيسية التي ستتعامل معها.

---

## الوكلاء

**الوكيل** هو مساعد ذكاء اصطناعي صوتي يمكنه التعامل مع المكالمات الهاتفية والمحادثات عبر الويب أو التفاعلات الصوتية داخل التطبيق.

<Tabs>
  <Tab title="بسيط">
    - **الشخصية** - كيف يبدو الوكيل وكيف يتصرف
    - **المعرفة** - المستندات والمعلومات التي يمكن للوكيل الرجوع إليها
    - **الصوت** - الصوت الذي يتحدث به
    - **اللغة** - اللغة(اللغات) التي يتحدثها
    - **التحية** - أول شيء يقوله للمتصلين
  </Tab>
  <Tab title="متقدم">
    - **أوامر النظام** - التعليمات التي تحدد سلوك الوكيل
    - **مزود LLM** - نموذج الذكاء الاصطناعي (Gemini، GPT-4، Claude)
    - **مزود STT** - محرك تحويل الكلام إلى نص (Deepgram، Google، AssemblyAI)
    - **مزود TTS** - صوت تحويل النص إلى كلام (ElevenLabs، Cartesia، Google)
    - **قاعدة المعرفة** - استرجاع مستندات مدعوم بـ RAG
    - **الوظائف** - استدعاءات أدوات مخصصة يمكن للوكيل تنفيذها
  </Tab>
</Tabs>

---

## خط أنابيب الصوت

كل محادثة صوتية تمر عبر خط أنابيب:

<Tabs>
  <Tab title="بسيط">
    ```
    المتصل يتحدث ← فورا تفهم ← الذكاء الاصطناعي يفكر ← فورا تجيب
    ```

    يحدث في الوقت الفعلي مع زمن استجابة أقل من 500 مللي ثانية.
  </Tab>
  <Tab title="متقدم">
    ```
    صوت الدخول → STT (Deepgram/Google) → LLM (Gemini/GPT-4) → TTS (ElevenLabs/Cartesia) → صوت الخروج
    ```

    يعالج خط الأنابيب الصوت في الوقت الفعلي باستخدام WebRTC عبر LiveKit. كل مزود يمكن تهيئته بشكل مستقل لكل وكيل.
  </Tab>
</Tabs>

---

## قاعدة المعرفة

قاعدة المعرفة تسمح لوكيلك بالإجابة على الأسئلة باستخدام مستنداتك الخاصة.

<Tabs>
  <Tab title="بسيط">
    ارفع ملفات (PDF، DOCX، TXT) أو الصق عناوين URL، ويمكن لوكيلك الرجوع إلى تلك المعلومات أثناء المحادثات.
  </Tab>
  <Tab title="متقدم">
    تستخدم قاعدة المعرفة RAG (الاسترجاع المعزز بالتوليد):
    1. يتم تقسيم المستندات وتحويلها إلى تمثيلات متجهية
    2. أثناء المحادثات، يتم استرجاع الأجزاء ذات الصلة بناءً على التشابه الدلالي
    3. يتم حقن السياق المسترجع في أوامر LLM
    4. يولد الوكيل استجابات مبنية على مستنداتك
  </Tab>
</Tabs>
