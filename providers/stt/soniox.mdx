# Soniox

Soniox offers enterprise-grade STT with custom vocabulary training and low latency.

---

## Overview

| Aspect | Details |
|--------|---------|
| Provider ID | `soniox` |
| Default Model | `precision` |
| Streaming | ✅ Native support |
| Latency | ~200-300ms |
| Languages | 15+ |

---

## Available Models

| Model | Accuracy | Latency | Best For |
|-------|----------|---------|----------|
| `precision` | 96%+ | ~300ms | Highest accuracy |
| `precision_ivr` | 95% | ~250ms | IVR systems |
| `speed` | 93% | ~150ms | Fastest response |

<Note>
**Recommended**: `precision` for general use, `speed` for ultra-low latency needs.
</Note>

## Configuration

### Basic Setup

```typescript
const agent = await vora.agents.create({
  name: 'Enterprise Agent',
  systemPrompt: 'You are a professional assistant.',
  stt: {
    provider: 'soniox',
    model: 'precision'
  }
});
```

### Full Configuration

```typescript
const agent = await vora.agents.create({
  name: 'Custom Agent',
  systemPrompt: 'You are a specialized assistant.',
  stt: {
    provider: 'soniox',
    model: 'precision',
    language: 'en-US',
    enableGlobalSpeakerDiarization: false,
    speakerChangeSensitivity: 0.5,
    customVocabularyId: 'vocab_medical_v1',
    profanityFilter: false
  }
});
```

## Why Soniox?

### 1. Custom Vocabulary Training

Train models on your domain vocabulary:

```typescript
// Upload custom vocabulary
const vocab = await soniox.createVocabulary({
  name: 'medical_terms',
  words: [
    { word: 'acetaminophen', pronunciation: 'ə.siː.tə.mɪn.ə.fən' },
    { word: 'cardiovascular', pronunciation: 'kɑː.di.oʊ.væs.kjʊ.lər' }
  ]
});

// Use in agent
stt: {
  provider: 'soniox',
  customVocabularyId: vocab.id
}
```

### 2. Enterprise Security

- SOC 2 Type II certified
- On-premise deployment available
- Data processing agreements
- Custom data retention

### 3. Low Latency

Optimized for real-time:
- ~150ms with speed model
- ~250ms with precision
- Consistent performance under load

Advanced speaker identification:

```typescript
stt: {
  provider: 'soniox',
  enableGlobalSpeakerDiarization: true,
  minNumSpeakers: 2,
  maxNumSpeakers: 4

```

## Custom Vocabulary

### Creating Vocabulary

Define domain-specific terms:

```typescript
const vocabulary = await soniox.createVocabulary({
  name: 'financial_terms',
  words: [
    {
      word: 'EBITDA',
      pronunciation: 'iː.bɪt.dɑː',
      displayAs: 'EBITDA'
    },
    {
      word: 'forex',

      word: 'cryptocurrency',
```

```typescript
stt: {
```

## Best Practices

<AccordionGroup>
  <Accordion title="Build Effective Vocabularies">
    Tips for custom vocabularies:

    1. **Include pronunciations** for unusual words
    2. **Add common misspellings** as alternatives
    3. **Specify display format** for abbreviations
    4. **Test with real audio** samples

    ```typescript
    {
      word: 'COVID-19',
      pronunciation: 'koʊ.vɪd.naɪn.tiːn',
      displayAs: 'COVID-19',
      alternatives: ['covid', 'coronavirus']
    }
    ```
  </Accordion>

  <Accordion title="Optimize Latency">
    For fastest response:

    ```typescript
    stt: {
      provider: 'soniox',
      model: 'speed',
      endpointSilenceMs: 500
    }
    ```

    Trade-off: Slightly lower accuracy for ~50ms faster response.
  </Accordion>

  <Accordion title="Speaker Diarization">
    For multi-party calls:

    ```typescript
    stt: {
      provider: 'soniox',
      enableGlobalSpeakerDiarization: true,
      speakerChangeSensitivity: 0.5,  // Default
      minNumSpeakers: 2,
      maxNumSpeakers: 5
    }
    ```

    Sensitivity:
    - Lower (0.3) = Fewer speaker changes detected
    - Higher (0.7) = More sensitive to speaker changes
  </Accordion>

  <Accordion title="Enterprise Deployment">
    For enterprise security requirements:

    - Request on-premise deployment
    - Configure data residency
    - Set up custom data retention
    - Enable audit logging
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Vocabulary Not Working">
    If custom terms aren't recognized:

    - Verify vocabulary ID is correct
    - Check pronunciation format (IPA)
    - Ensure vocabulary is published
    - Test with clear audio samples
  </Accordion>

  <Accordion title="Diarization Issues">
    If speakers aren't separated correctly:

    - Adjust speakerChangeSensitivity
    - Ensure distinct speaker voices
    - Check audio quality
    - Verify proper channel separation
  </Accordion>

  <Accordion title="Latency Problems">
    If responses are slow:

    - Switch to `speed` model
    - Check network latency
    - Reduce vocabulary size
    - Enable streaming mode
  </Accordion>
</AccordionGroup>
